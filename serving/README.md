# AudioMind serving

FastAPI service that loads the trained `best_model.keras` (10-genre GTZAN
classifier) and serves genre predictions over HTTP. This directory is a
self-contained Python project living alongside the `web/` (Next.js) frontend
in the AudioMind monorepo.

## Setup

Requires Python 3.12 (TensorFlow 2.17 does not ship wheels for 3.13+ at the
time of writing).

```bash
cd serving
python3.12 -m venv .venv
. .venv/bin/activate
pip install -r requirements.txt
```

## `norm_stats.json`

The model expects mel-spectrogram features normalized with the *training
set's* mean/std (the same values used in the Colab notebook at fit time).
`serving/norm_stats.json` currently holds the real values, computed in the
training notebook (`index.ipynb`) at cell 14 (`mean = X_train.mean()`,
`std = X_train.std()`) and printed in cell 27:

```json
{ "mean": -40.51021194458008, "std": 15.245159149169922 }
```

These were previously only ever saved into `gtzan_features.npz`
(Colab-only, not checked into git); they're now checked into this file
directly.

If the model is ever retrained, re-export the new mean/std from the
notebook and overwrite `serving/norm_stats.json`:

```python
import json
json.dump({"mean": float(mean), "std": float(std)},
          open("norm_stats.json", "w"), indent=2)
# then download norm_stats.json from the Colab kernel into serving/
```

## Running the API

The trained model must be present at the repo root as `best_model.keras`
(git-tracked; not regenerated by this scaffold).

```bash
cd serving
. .venv/bin/activate
uvicorn serving.app:app --reload
```

Run from the repository root so the `serving` package (and its
`serving.constants` / `serving.app` imports) resolves correctly.

## Tests

```bash
serving/.venv/bin/python -m pytest serving/tests/ -v
```

(or, with the venv activated and run from the repo root:
